# 🏭 Manufacturing Downtime Analysis

## 📌 Project Overview
This project focuses on analyzing **manufacturing downtime** to identify the main causes of production interruptions and improve operational efficiency.

The project combines **SQL, Python, and Data Visualization tools** to clean, analyze, and forecast downtime trends in manufacturing operations.

Currently, the project is in the **data cleaning and preprocessing stage using SQL**.

---

## 🎯 Project Objectives

- Analyze downtime patterns in manufacturing operations
- Identify the most common downtime causes
- Measure the impact of downtime on production efficiency
- Forecast future downtime using machine learning models
- Build a visualization dashboard to present key insights

## 📂 Dataset Structure

The dataset is organized as a **flat table**, where each row represents a machine observation during production.

| Column | Description |
|------|------|
| UDI | Unique identifier for each record |
| Product ID | Identifier of the manufactured product |
| Type | Product type (L, M, H) |
| Air temperature [K] | Air temperature during operation |
| Process temperature [K] | Machine process temperature |
| Rotational speed [rpm] | Speed of the machine |
| Torque [Nm] | Torque generated by the machine |
| Tool wear [min] | Tool wear duration |
| Target | Indicates if failure occurred (0 = No Failure, 1 = Failure) |
| Failure Type | Type of machine failure |

This **flat table structure** simplifies data analysis and allows direct use in **SQL queries, Python analysis, and machine learning models**.
